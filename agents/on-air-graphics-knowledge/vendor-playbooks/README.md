# vendor-playbooks/ — seeding convention

Holds per-vendor capability notes and trade-offs for broadcast-graphics engines.

**What belongs here:** Vizrt (Viz Engine/Artist/Pilot), Chyron, Ross (XPression),
Brainstorm, Singular.live, Unreal-for-broadcast — what each is strong at, where it
struggles, integration constraints, and licensing/operational notes.

**Each file needs:** the vendor, the capability claim, a `Last verified:` date (these
go stale fast), a `Source:` citation, and a confidence tag. Stay vendor-neutral — name
trade-offs honestly. After adding a file, bump `vendor-playbooks` in `INDEX.md`
`seed_counts`.
