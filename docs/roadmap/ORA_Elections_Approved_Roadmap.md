# ORA (Elections) — Approved Roadmap

> Source of truth for the ORA Elections module. Transcribed from the approved roadmap table.
> Spelling normalized where unambiguous; original terms preserved.

| # | Feature | Scope / Sub-features |
| --- | --- | --- |
| 1 | **Country election system setup / configuration** | Countries, Election systems (Presidency, Senate, Parliament), Chairs, Parties, Candidates, Rounds |
| 2 | **Calendar** | — |
| 3 | **Interactive Map** | Country Map, Districts / States, Candidates, Swing, Polling, Voting, Poll Close, Historical data, Sorting, Battleground Watch |
| 4 | **Entity Profile** | Candidates, Bio, Historical, Position, Related news, Trends, Entity mapping |
| 5 | **Breaking News** | Sentra Engine, API, Translation, lower-third |
| 6 | **Coalition** | Coalition updates, Coalition simulators, Promises, Companies, Local / Global |
| 7 | **Achievements** | Sectors (Economy, Social, Political, Military, Education, Industrial) |
| 8 | **Election preparations** | Campaigns, Swing Districts / States, Donations, Visits |
| 9 | **AI Prediction Engine** | Favor to win, Points, Turnout Intelligence |
| 10 | **Translation** | RTL + Arabic |
| 11 | **Latency** | ≤ 3 sec, Real time |
| 12 | **Broadcast** | Live Channels, Unified Intelligence Feed (Narratives, Intelligence coverage, Signals, Sentiments) |
| 13 | **Programs** | AI-generated programs / news list |
| 14 | **Take to Air** | Multi-selection, preview |
| 15 | **Integration** | Vizrt, Bloomberg, Reuters, AP, Edison Research, Decision Desk HQ |
| 16 | **Control Room** | Material approval, Happens on air |

## Notes
- **Translation (RTL + Arabic)** and **Latency (≤3s, real time)** are cross-cutting concerns that affect every feature, not standalone screens.
- This module ("ORA / Elections") is the first of several product areas; the app will be structured so additional modules can be added later.
- Phase 1 work is **UI with mock data** (see [Task — Breaking News Elections](../specs/Task_BreakingNews_Elections.md)).
